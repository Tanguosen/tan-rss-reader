


### 1. 用户与权限模块 (User & Auth Module)
**状态：已完成 (Completed)**
本模块负责系统的安全访问控制及用户全生命周期管理。

*   **后端实现**:
    *   **认证体系**: 基于 OAuth2 规范的 JWT (JSON Web Token) 认证流程，支持 Token 签发与验证。
    *   **用户管理**: 实现了标准的用户模型 (`User`)，支持注册、登录及个人信息获取。
    *   **权限控制**:
        *   区分 `admin` (管理员) 与 `user` (普通用户) 角色。
        *   管理员拥有专属 API (`/api/admin/*`)，具备更高权限。
        *   **安全保护**: 实现了防误操作逻辑，管理员无法删除自身账户或降级自身权限。
*   **前端实现**:
    *   **用户管理面板**: `AdminUsers.vue` 提供了完整的用户列表视图。
    *   **交互功能**: 支持在线修改用户角色（提权/降级）、账号状态控制（启用/禁用）以及账号删除。
    *   **登录/注册**: 完整的登录模态框交互，自动处理 Token 存储与过期跳转。
    *   **开发优化**: 暂时放宽注册密码校验规则（仅需6位字符，无大小写符号限制），便于开发测试。

### 2. 频道与订阅模块 (Channel & Subscription Module)
**状态：已完成 (Completed)**
本模块是用户发现内容和管理个人阅读列表的核心。

*   **频道广场 (Channel Square)**:
    *   **动态分类**: 支持后端动态下发频道分类，不再依赖前端硬编码。
    *   **搜索与发现**: 实现了全站频道的关键词搜索与分类浏览。
    *   **响应式布局**: Grid 布局已适配不同屏幕尺寸，提供一致的浏览体验。
*   **订阅关系**:
    *   **个性化订阅**: 用户可以订阅感兴趣的频道，系统自动维护 `Subscription` 关联表。
    *   **我的订阅**: 侧边栏实时同步显示用户已订阅的频道列表。

### 3. 信息源与内容模块 (Feed & Content Module)
**状态：稳定 (Stable) / 优化中**
本模块负责数据的采集、处理与展示，是系统的“心脏”。

*   **RSS 源管理 (Feed Management)**:
    *   **全生命周期管理**: 管理员可进行 RSS 源的增删改查 (CRUD)。
    *   **健康监控**: `AdminFeeds.vue` 提供可视化的源状态监控（红/绿状态灯），显示最后更新时间与错误信息。
    *   **更新策略**: 支持为不同源设置独立的更新频率 (`update_interval`)。
*   **内容抓取 (Data Fetching)**:
    *   **智能抓取**: `rss_fetcher.py` 实现了健壮的抓取逻辑，包含网络错误重试、HTTP 状态码处理。
    *   **特殊适配**: 针对 Arxiv 等特殊站点进行了 API 适配，绕过普通 RSS 的限制。
*   **阅读体验**:
    *   **文章列表**: 支持按时间、未读状态筛选文章。
    *   **详情视图**: 提供了可拖拽调整宽度的阅读面板，支持文章内容的富文本渲染。
    *   **移动端适配**: 窄屏模式下列表实现独立滚动，自动隐藏详情栏，点击文章直达全屏阅读页。

### 4. AI 增强模块 (AI Module)
**状态：已就绪 (Ready)**
本模块为阅读器提供智能化辅助功能。

*   **服务配置**:
    *   **多模型支持**: 支持配置兼容 OpenAI 格式的 AI 服务（如 Qwen, GLM 等）。
    *   **配置持久化**: AI 配置参数（API Key, Base URL）存储于数据库，支持在线热更新与连接测试。
*   **智能功能**:
    *   **智能摘要**: `Summarize` 接口可对长文章生成结构化摘要（一句话总结 + 关键点列表）。
    *   **多语言翻译**: 支持文章标题与正文的实时翻译，消除语言障碍。

### 5. 系统基础架构 (Infrastructure)
**状态：稳定 (Stable)**

*   **技术栈**:
    *   **后端**: Python FastAPI (高性能异步框架) + SQLAlchemy (ORM) + SQLite/PostgreSQL。
    *   **前端**: Vue 3 + Pinia (状态管理) + Electron (跨平台桌面应用外壳)。
*   **数据模型**: 数据库设计已规范化，包含 `feeds`, `entries`, `users`, `channels`, `ai_configs` 等核心表结构。
*   **API 设计**: 遵循 RESTful 规范，接口结构清晰，具备统一的错误处理机制。


### 6. 向量存储与聚类模块 (Vector Storage & Clustering Module)
**状态：已完成 (Completed)**
本模块构建了基于向量检索与聚类的智能分析系统。

*   **向量化 (Vectorization)**:
    *   **远程模型接入**: 集成 Qwen3-Embedding-0.6B 远程模型 (1024维)，实现高性能文本向量化。
    *   **自动处理**: RSS 数据流接入自动向量化任务，新文章入库即生成向量索引。
*   **存储与检索 (Storage & Retrieval)**:
    *   **Milvus 集成**: 接入远程 Milvus 向量数据库，使用 IVF_FLAT 索引与 COSINE 距离度量。
    *   **异步架构**: 采用 Async-to-Thread 模式处理向量库 I/O，避免阻塞主事件循环。
*   **聚类分析 (Clustering Analysis)**:
    *   **DBSCAN 引擎**: 实现基于密度的动态聚类算法，自动发现热点话题。
    *   **趋势分析**: 提供多时间维度（24小时/3天/1周）的话题聚合与展示。
*   **语义搜索 (Semantic Search)**:
    *   **自然语言检索**: 支持基于语义的模糊搜索，通过向量相似度匹配相关内容。
    *   **前端可视化**: 新增趋势分析视图 (`TrendsView`)，集成话题榜单与语义搜索界面。
