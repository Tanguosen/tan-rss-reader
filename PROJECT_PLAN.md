# 项目开发计划与进度文档

本文档详细列出了 TAN 项目的各模块开发计划、任务分解及预期产出物。

| 计划任务 | 类型 | 详细任务分解 | 预期产出物/里程碑 |
| :--- | :--- | :--- | :--- |
| **用户与权限模块开发** | 后端 | 1. **用户模型设计**: 定义 User 表结构 (ID, Username, Email, Role, PasswordHash)。<br>2. **认证机制实现**: 基于 OAuth2 的 JWT Token 签发与验证 (`auth.py`)。<br>3. **注册与登录接口**: 实现 `/auth/register` (含密码强度校验) 和 `/auth/login`。<br>4. **权限控制**: 实现 Admin/User 角色区分及接口访问控制 (`Depends(get_current_admin)`)。<br>5. **用户管理 API**: 管理员对用户的增删改查 (禁用/启用/提权)。 | 1. **用户体系上线**: 支持用户注册登录流程。<br>2. **安全认证**: 接口具备 Token 鉴权保护。<br>3. **管理后台**: 前端 `AdminUsers.vue` 可管理用户状态。<br>4. **开发便利性**: 实现了宽松的密码校验规则 (Dev 模式)。 |
| **频道与订阅模块开发** | 后端 | 1. **频道模型 (Feed)**: 存储 RSS 源元数据 (URL, Title, Icon)。<br>2. **订阅管理**: 实现订阅源的添加 (`/feeds/add`)、验证与去重。<br>3. **OPML 支持**: 实现 OPML 文件的导入与导出功能。<br>4. **自动刷新**: 集成 APScheduler 定时任务，周期性拉取更新。<br>5. **图标抓取**: 自动获取订阅源的 Favicon (`SiteIcon`)。 | 1. **订阅中心**: 用户可管理所有订阅源。<br>2. **数据同步**: 后台自动同步最新文章。<br>3. **迁移能力**: 支持从其他阅读器无缝迁移 (OPML)。 |
| **信息源与内容模块开发** | 后端 | 1. **文章模型 (Entry)**: 存储文章标题、内容、摘要、发布时间等。<br>2. **内容解析**: 使用 BeautifulSoup/FeedParser 清洗和提取正文。<br>3. **阅读状态**: 记录已读/未读、星标/收藏状态。<br>4. **全文检索**: 基于数据库或搜索引擎的关键词检索功能。<br>5. **流式加载**: API 支持分页与无限滚动加载。 | 1. **阅读体验**: 完整的内容流与文章详情页。<br>2. **状态同步**: 多端阅读进度保持一致。<br>3. **内容清洗**: 纯净的阅读视图，去除无关广告/样式。 |
| **AI摘要模块开发** | AI功能 | 1. **AI 服务集成**: 封装 LLM 调用接口 (OpenAI/Qwen 协议)。<br>2. **摘要生成逻辑**: 设计 Prompt 对文章进行摘要、关键词提取。<br>3. **异步处理**: 使用后台任务队列批量生成摘要，避免阻塞 API。<br>4. **持久化存储**: 创建 `EntryAI` 表存储生成结果，防止重复消耗 Token。<br>5. **容错处理**: 处理 JSON 解析失败与 API 超时情况。 | 1. **智能摘要**: 文章列表显示 AI 生成的精简摘要。<br>2. **成本优化**: 缓存机制减少 90% 重复 API 调用。<br>3. **多语言支持**: 自动翻译外文标题与摘要。 |
| **系统基础架构搭建** | 部署 | 1. **后端框架**: 基于 FastAPI 构建高性能异步 API 服务。<br>2. **数据库**: 选用 SQLite/PostgreSQL 配合 SQLAlchemy ORM。<br>3. **前端框架**: Vue 3 + Electron 构建跨平台桌面应用。<br>4. **外部服务集成**: 对接远程 Milvus (向量库) 与 Qwen (模型服务)。<br>5. **工程化配置**: 代码规范 (Linting)、环境变量 (.env) 管理。 | 1. **可运行制品**: 跨平台安装包 (macOS/Windows/Linux)。<br>2. **高可用架构**: 前后端分离，支持远程 AI 服务接入。<br>3. **开发文档**: 完善的启动脚本 (`start.sh`) 与依赖管理。 |
| **前端系统交互逻辑重设计规划** | 前端 | 1. **界面布局**: 采用经典的 三栏式布局 (侧边栏/列表/详情)。<br>2. **状态管理**: 使用 Pinia 管理全局状态 (用户/订阅/UI)。<br>3. **路由系统**: Vue Router 配置嵌套路由与守卫。<br>4. **交互优化**: 响应式设计、Loading 状态、错误提示 (Toast)。<br>5. **国际化 (i18n)**: 支持多语言界面切换 (中/英/日/韩)。 | 1. **现代化 UI**: 简洁美观的桌面客户端界面。<br>2. **流畅体验**: 无刷新页面切换与数据加载。<br>3. **多语言适配**: 完整的国际化支持。 |
| **开发信息详情页深度阅读辅助** | AI功能 | 1. **详情页增强**: `TopicDetailView` 集成 AI 分析面板。<br>2. **情感分析**: 可视化展示文章/话题的情感倾向 (正面/负面)。<br>3. **实体识别**: 自动提取并高亮文中的关键实体 (人名/地名/机构)。<br>4. **双语对照**: 提供原文与译文的对照阅读模式。<br>5. **相关推荐**: 基于向量相似度推荐相关阅读。 | 1. **深度阅读模式**: 超越传统 RSS 的阅读体验。<br>2. **智能辅助**: 帮助用户快速理解长文核心观点。<br>3. **知识关联**: 建立文章间的语义链接。 |
| **信息源向量化存储以及时序性处理** | AI功能 | 1. **向量化管道**: 自动将新文章标题/摘要转换为向量 (Embedding)。<br>2. **向量数据库**: Milvus 集合设计 (IVF_FLAT 索引, Cosine 距离)。<br>3. **历史回溯**: 开发脚本 `vectorize_history.py` 补全历史数据向量。<br>4. **时序处理**: 在向量检索中加入时间衰减或时间过滤 (`published_at`)。<br>5. **混合检索**: 结合关键词搜索与语义搜索。 | 1. **语义搜索引擎**: 支持自然语言描述的模糊搜索。<br>2. **数据资产**: 构建文章内容的向量知识库。<br>3. **时效性保证**: 搜索结果兼顾相关性与时效性。 |
| **探索开发其他AI功能** | AI功能 | 1. **RAG 问答**: 基于本地知识库的 "Chat with RSS" 功能。<br>2. **自动日报**: 每日自动生成个性化新闻简报。<br>3. **智能标签**: AI 自动为文章打标签分类。<br>4. **舆情监控**: 监控特定关键词的热度变化趋势。<br>5. **个性化推荐**: 基于用户阅读行为的协同过滤推荐。 | 1. **智能助手**: 内置的 AI 阅读伴侣。<br>2. **自动化报告**: 定期生成的行业/兴趣周报。<br>3. **千人千面**: 个性化的内容分发流。 |
| **话题趋势模块** | 前端,后端,AI功能,项目设计 | 1. **聚类算法**: 使用 DBSCAN 对向量空间数据进行聚类。<br>2. **趋势分析 API**: `/vector/cluster` 提供热点话题聚合。<br>3. **可视化看板**: `TrendsView` 展示话题气泡图/列表。<br>4. **时序演变**: 生成话题随时间发展的脉络图 (Timeline)。<br>5. **多维筛选**: 支持按时间范围 (24h/3天/1周/1月) 筛选趋势。 | 1. **趋势大屏**: 实时掌握关注领域的舆论热点。<br>2. **事件脉络**: 清晰展示事件的起因、经过与结果。<br>3. **洞察发现**: 发现由于信息过载而容易忽略的隐性关联。 |
